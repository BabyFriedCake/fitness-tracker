# ADW 自审提示词

只审查当前任务产生的未提交变更。

自审的目标，是在进入人工 Review 前，发现明确的实现错误、生命周期问题、
边界问题、持久化问题、导航问题和测试缺口。

## 审查范围

- 只使用当前 Agent。
- 不启动 subagent。
- 只审查当前 Task 修改的文件及其直接依赖的契约。
- 不重新设计功能，不扩大 Task Scope。
- 不执行 `git add`、`git commit`、`git push`、merge、tag 或 release。

## 必须检查

### 一、任务范围与架构

- Task 的全部验收标准是否已实现。
- 是否实现了 Task 明确禁止的 Non-goals。
- Route 和 Screen 是否保持轻量。
- SQL 是否进入 React、Route、Screen 或 UI 层。
- Domain、Application、Repository 和 Database 职责是否清晰。
- 是否修改了已经存在的历史 Migration。

### 二、数据与持久化

- Create 和 Update 是否正确保留生命周期字段。
- 普通更新是否错误覆盖 `status`、`archivedAt` 或 `createdAt`。
- 事务是否具备原子性。
- `BEGIN`、业务操作、`COMMIT` 和 `ROLLBACK` 的失败路径是否正确处理。
- 操作失败后是否可能留下部分写入的数据。
- 历史记录和共享的 Exercise 数据是否被意外修改。
- 删除模板动作时，是否只删除 TemplateExercise，而不会删除 Exercise。

### 三、异步生命周期

- 旧请求是否可能覆盖新请求结果。
- 组件卸载后是否仍可能更新 state。
- Loading 和 Error 状态中的重试操作是否真实有效。
- Save 是否读取最新草稿，而不是过期闭包。
- 重复点击 Save 是否可能产生重复写入。
- 异步加载尚未完成时，是否可能保存不完整数据。

### 四、导航与退出保护

- 自定义取消、系统返回和手势返回是否都被正确处理。
- 脏草稿是否必须确认后才能离开。
- 确认退出后是否会再次被导航拦截。
- 取消退出后是否残留过期的 pending navigation action。
- 保存成功后是否先关闭退出保护，再执行导航。
- 页面往返是否会产生重复 Route 实例。
- 从选择页返回后，是否可能丢失或覆盖当前草稿。

### 五、输入与边界

- 空值、非法值、负数、小数、`NaN` 和缺失值是否被处理。
- Route 参数是否正确支持预期的 `string | string[]`。
- 重复 ID 和缺失关联记录是否被明确处理。
- 排序和 position 是否稳定、连续。
- 归档记录是否保持只读，不能通过其他入口绕过限制。

### 六、测试质量

- 测试是否覆盖成功和失败路径。
- 是否覆盖重试、重复操作、异步竞态和导航生命周期。
- 是否验证了真实状态协作，而不只是 mock 回调调用次数。
- 测试是否为了通过而固化了错误行为。
- 是否存在实现已经改变，但测试仍在验证旧行为的情况。

## 执行规则

- 实现期间优先运行当前任务的定向测试。
- Self Review 只执行一轮。
- 自动修正最多执行一轮。
- 修正后只重新运行受影响的定向测试。
- 完整的 `format`、`lint`、`typecheck` 和 `test`
  只在实现和 Self Review 完成后运行一次。
- 一轮修正后仍存在阻塞问题时，停止并报告。
- 不允许继续自动循环修复。

## 最终报告

Final Report 不超过 50 行，并包含：

- 自审发现
- 已完成修正
- 定向测试
- 最终验证
- Repository Hygiene
- Ready for Human Review
