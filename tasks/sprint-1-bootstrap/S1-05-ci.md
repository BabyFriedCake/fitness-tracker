# S1-05 — Add GitHub Actions CI

## Goal

Run repository validation automatically for pushes and pull requests.

## Read first

- `AGENTS.md`
- `docs/08-Development/testing-strategy.md`
- `docs/08-Development/git-workflow.md`

## Scope

Create a GitHub Actions workflow that:

1. Checks out the repository
2. Installs the supported Node and pnpm versions
3. Uses dependency caching
4. Runs `pnpm install --frozen-lockfile`
5. Runs format check
6. Runs lint
7. Runs typecheck
8. Runs tests

Do not add EAS Build or App Store deployment yet.

## Acceptance criteria

- [ ] Workflow runs on pull requests and pushes to `main`.
- [ ] A dependency or type error fails CI.
- [ ] A test failure fails CI.
- [ ] The workflow uses the committed lockfile.
- [ ] No credentials or secrets are required.
- [ ] README or development docs list the validation commands if they changed.

## Suggested commit

```text
ci: validate mobile workspace
```
