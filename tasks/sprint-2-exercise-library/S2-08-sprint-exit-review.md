# S2-08 — Sprint 2 Exit Review

Execution Prompt: `tasks/prompts/sprint-exit-review.md`

## Goal

Review Sprint 2 as a complete Exercise Library increment and determine whether the project is ready for Sprint 3.

## Scope

Review:

- Exercise domain contracts
- Seed import
- SQLite repository
- Browse list
- Search and filters
- Detail
- Selection contexts
- Documentation and roadmap
- CI and repository hygiene
- Offline behavior and data-source licensing

Do not:

- Implement new features
- Modify source code during the initial review
- Begin Workout Template development
- Hide known technical debt

## Acceptance Criteria

- [ ] All Sprint 2 tasks are completed or explicitly documented as incomplete.
- [ ] Exercise Library matches P007.
- [ ] Search/filter behavior is correct and tested.
- [ ] Data import is licensed, deterministic, validated, and idempotent.
- [ ] SQL remains isolated in database infrastructure.
- [ ] Browse and selection modes are both tested.
- [ ] CI passes on GitHub.
- [ ] Roadmap is synchronized.
- [ ] `docs/09-Release/Sprint-2-Exit-Report.md` is generated from the report template.
- [ ] Ready-for-Sprint-3 decision is evidence-based.

## Expected Report Metrics

Include at minimum:

- Tasks completed
- Test suites/tests
- Seed exercise count
- Search/filter coverage
- Database migrations
- Architecture violations
- Repository hygiene
- Known technical debt
- Native-device verification status
