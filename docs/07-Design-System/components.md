# Fitness Tracker Components

Version: v1.0  
Status: Approved  
Last Updated: 2026-07-14

---

## 1. Button

### Primary Button

用于页面唯一主操作。

示例：

- 开始训练
- 暂停训练
- 继续训练
- 保存模板
- 保存并完成

要求：

- 高度至少 52
- 宽度尽量占满内容区
- 文字清晰
- 支持 loading 和 disabled
- 点击区域不得小于视觉区域

### Secondary Button

用于：

- 修改
- 查看详情
- 添加动作
- 继续

### Destructive Button

用于：

- 放弃训练
- 删除模板
- 清除全部数据

必须使用明确文案，不只显示垃圾桶图标。

### Text Button

用于低优先级操作：

- 跳过
- 取消
- 稍后设置

---

## 2. Number Stepper

用于重量、次数和时间调整。

示例：

```text
[-2.5]  80.0  [+2.5]
```

要求：

- 中间数值可点击编辑
- 两侧按钮点击区域至少 44 × 44
- 长按是否连续调整可放到后续版本
- 调整后立即显示新值
- 不允许出现非法负数

### 重量

- 默认步进来自 UserSetting
- V1 默认建议 2.5 kg
- 支持小数

### 次数

- 默认步进 1
- 只允许非负整数

---

## 3. Text Input

用途：

- 模板名称
- 模板描述
- 训练备注
- 搜索动作

状态：

- default
- focused
- error
- disabled

错误信息必须显示在输入框附近。

---

## 4. Card

### Template Card

显示：

- 模板名称
- 训练部位
- 最近训练时间
- 动作数量

### Workout History Card

显示：

- 日期
- Session 名称
- 时长
- 组数
- 总容量
- 完成率

### Recommendation Card

显示：

- 动作
- 上次表现
- 建议挑战
- 原因

建议卡片不得表现为强制命令。

---

## 5. List Row

用于：

- 动作列表
- 设置列表
- 历史列表

标准结构：

```text
左侧图标或序号
主标题
副标题
右侧状态或箭头
```

---

## 6. Status Chip

用于：

- 正常
- 疲劳
- 经期
- 不适
- 草稿
- 已完成
- 已取消

要求：

- 文字必须存在
- 颜色仅作辅助
- 可选状态应有明确选中效果

---

## 7. Timer Display

用于休息倒计时。

要求：

- 大号等宽数字
- 格式固定为 `MM:SS`
- 不显示负数
- 暂停时有清晰状态
- 到期时显示 `00:00` 和“休息结束”

---

## 8. Progress Indicator

训练执行页显示：

- 当前动作 / 总动作
- 已完成组 / 总目标组

V1 优先文字和简单进度条。

不使用复杂圆环动画。

---

## 9. Modal / Bottom Sheet

用途：

- 编辑重量
- 自定义休息时间
- 结束训练确认
- 删除确认

原则：

- 重要确认使用 Modal
- 快速编辑使用 Bottom Sheet
- 破坏性操作必须明确结果
- 不允许通过点击背景直接执行危险操作

---

## 10. Empty State

组成：

- 简短标题
- 一句话说明
- 一个明确操作

示例：

```text
还没有训练模板

创建第一个模板，开始记录训练。

[创建训练模板]
```

---

## 11. Toast / Inline Feedback

适合：

- 已保存
- 已添加动作
- 计时已延长 30 秒

不适合：

- 数据写入失败
- 训练恢复失败
- 权限被拒绝

严重错误必须使用持久化提示。
