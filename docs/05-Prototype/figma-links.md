## Implementation Notes

Codex should use this prototype only for:

- visual layout reference
- interaction expectation
- screen hierarchy
- UI consistency review

Codex should not:

- generate replacement architecture from Figma
- copy generated Figma code
- modify domain models based only on visual design

---

## Review Workflow

When implementing UI changes:

1. Check related Prototype specification markdown.
2. Check PRD requirements.
3. Check Architecture constraints.
4. Compare implementation with Figma visual reference.
5. Update documentation if behavior changes.
