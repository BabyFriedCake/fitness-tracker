Build Fitness Tracker according to the project specifications.

1. docs/02-Constitution/constitution.md

4. docs/05-Prototype/

## Golden Rules

- Specification before implementation.
- Keep Single Source of Truth.
- Do not duplicate documents.
- Do not change product behavior without updating Prototype.
- Do not change data structure without updating Database documents.
- Keep Domain Model stable.
- Prefer refactoring over adding technical debt.
- Update documentation together with code.

## Coding Principles

- Small commits
- Clear commit messages
- Keep components reusable
- Prefer composition over duplication

## Pull Request Checklist

- Product behavior unchanged or documented.
- Prototype updated if UI changed.
- Database docs updated if schema changed.
- Tests added or updated.
